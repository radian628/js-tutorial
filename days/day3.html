Day 3

DRY

There is a principle in programming that you will hear me frequently allude to and praise to no end. That is the principle of "Don't Repeat Yourself" (DRY for short), the idea that code should be free of pointless repetition.

Writing DRY code is important because doing so drastically aids in maintainability. For example, let's say you have several places where you want apply a complex formula to a number. You type out the formula each time. But then the formula turns out to have a mistake in it, and you have to go and change every single copy you just made to fix it. If you had made this formula a function, you'd only have to fix the mistake once, inside of the function body.

Logical Operators

Checking Multiple Conditions

Consider the following situation: A roller coaster requires its passengers to be at least 120 centimeters tall, 40 kilograms in weight, and 10 years old. You make a function to check whether someone can ride the roller coaster with a given height, weight, and age. You could do it like this...


function canRideRollerCoaster(height, weight, age) {
    if (height >= 120) {
        if (weight >= 40) {
            if (age >= 10) {
                console.log("You can ride the roller coaster!");
            } else {
                console.log("You can't ride the roller coaster!");
            }
        } else {
            console.log("You can't ride the roller coaster!");
        }
    } else {
        console.log("You can't ride the roller coaster!");
    }
}

Using Logical Operators

However, anyone who adheres to the DRY principle will recoil in disgust upon seeing the sheer amount of repetition in the above code. Look at how the "You can't ride the roller coaster!" is pointlessly repeated three times!

Of course, we are at the mercy of the syntax of if-statements here. Multiple if-statements can't share the same else block. If only there was a way to combine all of these if-statements into a single one, checking to see whether all of the conditions are true?...

Oh wait, there is. This is a job for the logical operators, which essentially combine or manipulate booleans. They can express whether all conditions are true, whether any conditions are true, or can negate a condition. The operator we need here is the && operator, which only evaluates to true if both of its operands are true. Here's how it works:

Expression Result
true && true true
true && false false
false && true false
false && false false

And here's how we could use it to fix the following example. Before you reveal it, try and implement the improved version yourself:


    function canRideRollerCoaster(height, weight, age) {
        if (height >= 120 && weight >= 40 && age >= 10) {
            console.log("You can ride the roller coaster!");
        } else {
            console.log("You can't ride the roller coaster!");
        }
    }

&&, ||, and !

Another logical operator, ||, returns true provided that any of its operands are true. A third one, !, simply switches the following boolean from true to false, or vice-versa. This table demonstrates how they work:

Expression Result Expression Result Expression Result
true && true true true || true true !true false
true && false false true || false true !false true
false && true false false || true true
false && false false false || false false

The && operator is also called "AND"; || is called "OR"; and ! is called "NOT".

Objects

What are Objects?

The idea of an object may just be the most important concept in JavaScript. Objects are- put simply- a bunch of variables "within" another variable. They are made up of a list of keys, strings that act as labels; and values, pieces of data attached to the keys. A key and its corresponding value together is called an entry, a property, or a key-value pair. Yes, all these names do, in fact, refer to the same thing.

Creating Objects

Consider the following data, which describe a product at a grocery store- potato chips- of which the store has 123 bags- that sell for $1.25:

Key Value
Name Potato Chips
Price 1.25
Quantity In Stock 123

As you can see, these data form a "key-value" relationship (and yes, "data" is plural). They can be turned into a JavaScript object like so:


let product = {
    name: "Potato Chips",
    price: 1.25,
    quantityInStock: 123
};

As you can see, objects are assigned to variables using let, just like many of the other data types. They are surrounded by curly brackets. Between the curly brackets, entries are separated with commas, and keys are separated from their values with colons.

Be aware of the naming conventions of keys. Keys generally have no spaces in their names. Every word but the first should be capitalized. This also applies to JavaScript functions and variables.

Accessing Properties

The value of an object's property can be accessed by typing in the name of the object, followed by a . (period), and then the name of that property's key. For instance, product.name is "Potato Chips".

You can also change and add properties to an object. Let's say the store wants to open a new branch in the UK, where they call chips "crisps". We could use the following line of code to update the name.


product.name = "Potato Crisps";

We can also add new properties by simply setting a property that doesn't exist yet. Let's say the store wants to indicate that the chips are on sale. They could do that as follows:


product.onSale = true;

The onSale property never existed before, but it does now!

What is console.log?

All that being said, what is console? What is log? And what is console.log?

  • console is an object.
  • log is the name of one of console's keys.
  • console.log represents the value that corresponds to the log key. It might seem odd that this value is a function, but remember, functions are data!

Return Values

Many functions do more than just execute code- they can also "return" a value. This is where a call to a function acts like a piece of data. Return values are most frequently used to show what a function outputs. A function can return a value by putting the keyword return before that value. Note that the return keyword also prematurely ends the execution of the code in the function. Here's an example of return statements in action:


function addFiveTo(num) {
    return num + 5;
}

console.log(addFiveTo(6));
            

11

Note that- unlike in previous examples, where our functions called console.log to produce output; this function- when called- simply evaluates to a number.

HTML-JavaScript Interaction

document.getElementById

With all of this explained, we can combine HTML and JavaScript together... sort of. Consider the following HTML and JavaScript in their respective tabs:


<!DOCTYPE html>
<html>
    <head>
    
    </head>
    <body>
        <p id="paragraph">JavaScript will change this text!</p>
    </body>
</html>
            

let paragraph = document.getElementById("paragraph");
paragraph.innerText = "new text";

This new function document.getElementById returns a JavaScript object representing an HTML element. More specifically, the JavaScript object represents an HTML element with a given id attribute. In the above example, we're getting the element with the id paragraph, which the p element in the body has.

In the second line, we change the value of the element's innerText property to "new text". The innerText property represents the text inside of an HTML element, and- when changed- the HTML element matches to update it. As such, all of this code together outputs the following:

new text

We can also read these properties just like any other property of a JavaScript object. So if we added console.log(paragraph.innerText) after our current code, the program would then output "new text" to the console.

Event Listeners

We had JavaScript control HTML in the last section. Now let's do the reverse with event listeners! Event listeners are simply functions that the browser calls whenever some event happens- for example if an element is clicked, the user scrolls the mouse, or a key is pressed. Here is an example:


<!DOCTYPE html>
<html>
    <head>
    
    </head>
    <body>
        <button id="click-me-button">Click me!</button>
    </body>
</html>
            

function clickHandler() {
    console.log("clicked!");
}

let button = document.getElementById("click-me-button");
button.onclick = clickHandler;

The HTML is standard stuff- a button with an id of click-me-button. The JavaScript starts with a function declaration that simply logs "clicked!" to the console. Then, we get the button with document.getElementById (notice, again, how the parameter to document.getElementById and the id attribute of the element are identical). Finally, we set the button's onclick property to the function. The browser knows to run the onclick function (if it exists) whenever the element is clicked, so clicking the button will now log "clicked!" to the console.

Note that the JavaScript can also be written like this to set button.onclick directly to the function without an intermediary. Also note the lack of a function name:


let button = document.getElementById("click-me-button");

button.onclick = function() {
    console.log("clicked!");
}
