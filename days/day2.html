Introduction

Programming Languages

Welcome back! Let's get started: JavaScript is a programming language. But what even is a programming language?

A programming language is- put simply- a list of instructions executed by a computer. Unless you tell the computer to do otherwise, these instructions will appear to execute in sequence, one after the next. They can do a variety of things, though most of it boils down to manipulating data, inputting data, outputting data, and deciding when to manipulate data.

Applications of JavaScript

As you probably know by now, JavaScript (JS for short) is mainly used to make web pages interactive. There's even JavaScript on this page! Among other things, it creates that useful navigation section you see on the left, preventing me from having to spend time copying all of the text in the headers over there (and subsequently having to update them all if I want to make any changes).

Variables

Hello World Revisited

Let's change up our "Hello World" code a bit:

let variable = "Hello World";
console.log(variable);

Type this into JSFiddle, and see what happens. It should output the same thing as the first "Hello World" program. What you've actually done here is create a variable. Let's analyze the first line:

let variable = "Hello World";

The let keyword creates, or declares a variable. The text after let is the name of the variable. In the example above, the variable's name is variable, though it could be any string of letters (no spaces though!). Next, we set it equal to the string Hello World with the = "Hello World"; part. Remember, every line in JavaScript should end with a semicolon. It will still work without the semicolons, but this can lead to unexpected behavior.

Now for the second line:

console.log(variable);

When you write the name of a variable somewhere, it represents the underlying value of the variable. That's why console.log(variable); prints "Hello World", not variable. It's just like how in math, you could set a variable x equal to 5. Then, wherever you see x, you can treat it as if it's just the number 5. Speaking of numbers...

Numerical Variables

Variables can be numbers too!

let x = 5;
console.log(x);

As you might have expected, this will print the following to the console:

5

Math!

You can do math on both numbers and variables with numerical values. Consider the following example:

let y = 7;
y = y + 3;
console.log(y);

Here's the output:

10

Note that using single-letter variable names here is arbitrary. The type of data a variable represents has no influence on the name it can have. Another thing to note is that variables can be assigned and reassigned with the = operator (operators are just symbols that act on pieces of data). Remember to only use let when you define a variable for the first time. You can omit it for all subsequent reassignments of the variable.

Order of Operations (PEMDAS)

PEMDAS applies in JavaScript. In other words, statements within parentheses execute first. Without parentheses, multiplication and division happen before addition and subtraction. The following code outputs 14.

console.log(10 + 8 / 2);

Conversely, this code outputs 9.

console.log((10 + 8) / 2);

Booleans

What are Booleans?

Booleans are another data type in JavaScript (a data type is a distinct form that data (i.e. variables) can take). Booleans can only take one of two values: true or false.

let boolean = true;
let anotherBoolean = false;

Booleans from Comparison Operators

Comparison operators are operators that- as you might expect- compare the size of two numbers. They output booleans- true if the comparison is accurate, and false if it is not. As an example, 6 > 2 is true (because six is, in fact, greater than two), whereas 5 < 4 is false (because five is not less than four).

Comparison Operator Table

This might look a bit intimidating. However, if you're well-versed with comparison from your math class, you'll likely only need to consider the leftmost two rows (the others are just examples).

Operator Name Example 1 Output 1 Example 2 Output 2

< Less Than 6 < 3 false 4 < 4 false
>= Greater Than or Equal To 6 >= 3 true 4 >= 4 true
<= Less Than or Equal To 6 <= 3 false 4 <= 4 true
== Equal 6 == 3 false 4 == 4 true
!= Not Equal 6 != 3 true 4 != 4 false

Warning: Do not confuse == (equals) with = (assignment). The former tests whether two things are equal (outputting true if they are, and false otherwise), while the latter sets the variable on the left to the value on the right.

If-Statements

If-statements allow you to run code conditionally- in other words, they only run the code within them if a given boolean is true. For example, the code below only prints "x is greater than 7!" if the variable x is greater than 7.

let x = 10;
if (x > 7) {
    console.log("x is greater than 7!");
}
x = 5;
if (x > 7) {
    console.log("x is greater than 7!");
}

The way the code is currently structured, how many times will it output "x is greater than 7!"? Which if-statement(s) will run?

Answer:

Once. Only the first if-statement will execute the code between its curly braces.

Else

Take the following example:

let chanceOfRain = 0.75;
if (chanceOfRain > 0.5) {
    console.log("It is likely to rain.");
}
if (chanceOfRain <= 0.5) {
    console.log("It is unlikely to rain.");
}

You may quickly realize that the second if-statement will only run if the first one doesn't. After all, the first one uses a "greater than" condition, whereas the second uses a "less than or equal to" condition- both with the same operands (operands is a term for what operators operate on)! The else keyword can simplify this code- it lets you provide a block of code that will only run if the preceding if-statement does not.

The following example functions identically to the previous one, but uses an else block instead of another if-statement. To use else, first type the word else, followed by curly brackets, in which you'll put your code. Unlike the if-statement, you don't need parentheses.

let chanceOfRain = 0.75;
if (chanceOfRain > 0.5) {
    console.log("It is likely to rain.");
} else {
    console.log("It is unlikely to rain.");
}

Side note: Changing the structure of code while keeping the same functionality (which is what we did here) is called refactoring.

Else If

Multiple if-statements can be chained together using else if. else if functions identically to else, except the code after the else if only runs if the supplied condition is true. Multiple else ifs can even be chained together, each one only executing if all the previous ones have not.

To use else if, just type else followed by an if-statement just like you'd write it on your own.

Consider the following example, which does exactly what you might expect:

let chanceOfRain = 0.2;
if (chanceOfRain > 0.8) {
    console.log("It is likely to rain.");
} else if (chanceOfRain > 0.3) {
    console.log("It is somewhat likely to rain.");
} else {
    console.log("It is unlikely to rain.");
}

Given the above code, what is the likelihood of rain? Check your answer by typing the example into JSFiddle.

Another Example of Else-if

Edit the previous example so that it prints "It's definitely not going to rain!" if the chance of rain is exactly 0. Test your code with different values for chanceOfRain to see if it works! let chanceOfRain = 0.2;

Functions

Let's say we have to check the chance of rain three different times, using the previous code we wrote. Let's represent the three chances we're interested in using variables:

let chanceOfRain1 = 0;
let chanceOfRain2 = 0.5;
let chanceOfRain3 = 0.9;

We could, of course, do it like this:

let chanceOfRain1 = 0;
let chanceOfRain2 = 0.5;
let chanceOfRain3 = 0.9;

if (chanceOfRain1 > 0.8) {
    console.log("It is likely to rain.");
} else if (chanceOfRain1 > 0.3) {
    console.log("It is somewhat likely to rain.");
} else if (chanceOfRain1 != 0) {
    console.log("It is unlikely to rain.");
} else {
    console.log("It's definitely not going to rain!");
}

if (chanceOfRain2 > 0.8) {
    console.log("It is likely to rain.");
} else if (chanceOfRain2 > 0.3) {
    console.log("It is somewhat likely to rain.");
} else if (chanceOfRain2 != 0) {
    console.log("It is unlikely to rain.");
} else {
    console.log("It's definitely not going to rain!");
}

if (chanceOfRain3 > 0.8) {
    console.log("It is likely to rain.");
} else if (chanceOfRain3 > 0.3) {
    console.log("It is somewhat likely to rain.");
} else if (chanceOfRain3 != 0) {
    console.log("It is unlikely to rain.");
} else {
    console.log("It's definitely not going to rain!");
}

However, as you can probably see, this is horribly repetitive. Imagine if you wanted to add a condition that says "It will certaintly rain!" if the chance of rain is 1? You'd have to do that three times!

Because of this (along with their general laziness), programmers hate needless repetition, and thus, they've developed tools to combat it. In fact, if any of your code is extremely repetitive, you should seek out a better way, because a disgruntled programmer has likely already thought of one, and added it to the language.

Often the best way of eliminating repetition is using functions. A function is simply a named block of code that you can run as many times as you want. They are created with the keyword function, followed by the function name (which works the same as a variable name), after which you put parentheses, and finally curly brackets in which you put your code. Here's a function that simply writes "Hello World!" to the console:

function helloWorld() {
    console.log("Hello World!");
}

Between the parentheses, you can optionally put a list of function arguments. These are just a bunch of pieces of data that you "pass" or "plug" into the function, influencing its behavior. In math class, if you've seen something like f(x), the function argument is basically the x.

To actually run the function, type the name of the function, followed by a set of parentheses. Between the parentheses, put the values you want the arguments to take. Even if there are no arguments, you should still include the parentheses! In JavaScript, functions are just another type of data (you'll hear me repeat this a lot), so just writing the function name merely refers to the function itself!

Here's the previous example, refactored to use a function:

function describeChanceOfRain(chanceOfRain) {
    if (chanceOfRain > 0.8) {
        console.log("It is likely to rain.");
    } else if (chanceOfRain > 0.3) {
        console.log("It is somewhat likely to rain.");
    } else if (chanceOfRain != 0) {
        console.log("It is unlikely to rain.");
    } else {
        console.log("It's definitely not going to rain!");
    }
}

let chanceOfRain1 = 0;
let chanceOfRain2 = 0.5;
let chanceOfRain3 = 0.9;

describeChanceOfRain(chanceOfRain1);
describeChanceOfRain(chanceOfRain2);
describeChanceOfRain(chanceOfRain3);

Props to you if you figured this out already, but our good old friend console.log is a function too!

Challenge: N Hello Worlds

Here's a fun challenge: Create a function helloWorlds that prints Hello World! n times, where n is one of its arguments. So, for example, helloWorlds(4); would print the following:

"Hello World!"
"Hello World!"
"Hello World!"
"Hello World!"

Stuck? Here's a hint:

Hint: helloWorlds should call itself.

Possible solution:

function helloWorlds(n) {
    if (n > 0) {
        console.log("Hello World!");
        helloWorlds(n - 1);
    }
}

The phenomenon of a function calling itself is called recursion. A function that does recursion is called a recursive function.
