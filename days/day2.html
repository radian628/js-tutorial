Introduction

Programming Languages

Welcome back! Let's get started: JavaScript is a programming language. But what even is a programming language?

A programming language is- put simply- a list of instructions executed by a computer. Unless you tell the computer to do otherwise, these instructions will appear to execute in sequence, one after the next. They can do a variety of things, though most of it boils down to manipulating data, inputting data, outputting data, and deciding when to manipulate data.

Applications of JavaScript

As you probably know by now, JavaScript (JS for short) is mainly used to make web pages interactive. There's even JavaScript on this page! Among other things, it creates that useful navigation section you see on the left, preventing me from having to spend time copying all of the text in the headers over there (and subsequently having to update them all if I want to make any changes).

Variables

Hello World Revisited

Let's change up our "Hello World" code a bit:

let variable = "Hello World";
console.log(variable);

Type this into JSFiddle, and see what happens. It should output the same thing as the first "Hello World" program. What you've actually done here is create a variable. Let's analyze the first line:

let variable = "Hello World";

The let keyword creates, or declares a variable. The text after let is the name of the variable. In the example above, the variable's name is variable, though it could be any string of letters (no spaces though!). Next, we set it equal to the string Hello World with the = "Hello World"; part. Remember, every line in JavaScript should end with a semicolon. It will still work without the semicolons, but this can lead to unexpected behavior.

Now for the second line:

console.log(variable);

When you write the name of a variable somewhere, it represents the underlying value of the variable. That's why console.log(variable); prints "Hello World", not variable. It's just like how in math, you could set a variable x equal to 5. Then, wherever you see x, you can treat it as if it's just the number 5. Speaking of numbers...

Numerical Variables

Variables can be numbers too!

let x = 5;
console.log(x);

As you might have expected, this will print the following to the console:

5

Math!

You can do math on both numbers and variables with numerical values. Consider the following example:

let y = 7;
y = y + 3;
console.log(y);

Here's the output:

10

Note that using single-letter variable names here is arbitrary. The type of data a variable represents has no influence on the name it can have. Another thing to note is that variables can be assigned and reassigned with the = operator (operators are just symbols that act on pieces of data). Remember to only use let when you define a variable for the first time. You can omit it for all subsequent reassignments of the variable.

Order of Operations (PEMDAS)

PEMDAS applies in JavaScript. In other words, statements within parentheses execute first. Without parentheses, multiplication and division happen before addition and subtraction. The following code outputs 14.

console.log(10 + 8 / 2);

Conversely, this code outputs 9.

console.log((10 + 8) / 2);

Functions

Functions are essentially reusable blocks of code. You can create one with the keyword function followed by the name of the function, a pair of parentheses, and finally curly brackets, between which you put the function's code. This is called declaring a function, and the code within the function is the function body. This function below just logs "Hello World!" to the console.


function helloWorld() {
    console.log("Hello World!");
}

In JavaScript, functions are data and therefore can be assigned to variables. As such, you could also write the above example like this (though the former is still more common):


let helloWorld = function () {
    console.log("Hello World!");
}

Calling Functions

In order to actually run the code within a function, you have to call it. To call a function, type the name of the function, followed by a pair of parentheses. To call the helloWorld function above, you would need to type the following:


helloWorld();

Remember to include the parentheses. Since functions are data, just typing the function name (helloWorld) refers to the function itself.

Function Parameters

Data can be "passed" into a function to modify the way it works. In the function declaration, you can put a comma-separated list of names to represent the parameters the function accepts. Consider the following function that converts fahrenheit to celsius:


function fahrenheitToCelsius(degreesInFahrenheit) {
    console.log((degreesInFahrenheit - 32) * 5 / 9);
}

This function accepts one parameter, degreesInFahrenheit. Inside the function body, we can treat degreesInFahrenheit like a variable. When calling a parameterized function, we must give actual values to the parameters. We supply these parameters as a comma-separated list within the parentheses in the function call. This example calculates 70 degrees fahrenheit in celsius, and logs it to the console:


fahrenheitToCelsius(70);

However, we can make this function even better...

Return Values

Just console.log()ing the degrees in celsius doesn't make much sense- after all, what if we want to do something else with it? We could, of course, use a variable to store the value:


let degreesInCelsius;

function fahrenheitToCelsius(degreesInFahrenheit) {
    degreesInCelsius = (degreesInFahrenheit - 32) * 5 / 9;
}

Avoid doing this if at all possible, because there is actually a much better method of gaining access to the output of a function: return values. When a function "returns" a piece of data, a call to that function acts as if it is that data. A function can return some data by simply putting the keyword return before the data. Let's make the fahrenheitToCelsius function return the temperature in celsius, so we can use it elsewhere:


function fahrenheitToCelsius(degreesInFahrenheit) {
    return (degreesInFahrenheit - 32) * 5 / 9;
}

We could now, for example, store the output of this function in a variable, and do whatever we want with it:


let seventyInCelsius = fahrenheitToCelsius(70);
let seventyInKelvin = seventyInCelsius + 273.15;

Notice how fahrenheitToCelsius(70) is being treated as a piece of data- this is only possible because of the return statement.

Note that- when a function encounters a return statement- the function will end prematurely. In addition, functions that don't return anything will return the value undefined, which is another data type you'll see often.


function test() {
    console.log("this code will run!");
    return;
    console.log("but this won't!");
}

The above function will return undefined because there is nothing after the return statement. If there were no return statement at all, it would also return undefined.

DRY

There is a principle in programming that you will hear me frequently allude to and praise to no end. That is the principle of "Don't Repeat Yourself" (DRY for short), the idea that code should be free of pointless repetition.

Writing DRY code is important because doing so drastically aids in maintainability. For example, let's say you have several places where you want apply a complex formula to a number. You type out the formula each time. But then the formula turns out to have a mistake in it, and you have to go and change every single copy you just made to fix it. If you had made this formula a function, removing the repitition, you'd only have to fix the mistake once, inside of the function body.
