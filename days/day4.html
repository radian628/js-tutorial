Day 4

Objects

What are Objects?

The idea of an object may just be the most important concept in JavaScript. Objects are- put simply- a bunch of variables "within" another variable. They are made up of a list of keys, strings that act as labels; and values, pieces of data attached to the keys. A key and its corresponding value together is called an entry, a property, or a key-value pair. Yes, these three names do, in fact, refer to the same thing.

Creating Objects

Consider the following data, which describe a product at a grocery store- potato chips- of which the store has 123 bags- that sell for $1.25:

Key Value
Name Potato Chips
Price 1.25
Quantity In Stock 123

As you can see, these data form a "key-value" relationship (and yes, "data" is plural). They can be turned into a JavaScript object like so:


let product = {
    name: "Potato Chips",
    price: 1.25,
    quantityInStock: 123
};

As you can see, objects are assigned to variables using let, just like many of the other data types. They are surrounded by curly brackets. Between the curly brackets, entries are separated with commas, and keys are separated from their values with colons.

Be aware of the naming conventions of keys. Keys generally have no spaces in their names. Every word but the first should be capitalized. This also applies to JavaScript functions and variables.

Accessing Properties

The value of an object's property can be accessed by typing in the name of the object, followed by a . (period), and then the name of that property's key. For instance, product.name is "Potato Chips".

You can also change and add properties to an object. Let's say the store wants to open a new branch in the UK, where they call chips "crisps". We could use the following line of code to update the name.


product.name = "Potato Crisps";

We can also add new properties by simply setting a property that doesn't exist yet. Let's say the store wants to indicate that the chips are on sale. They could do that as follows:


product.onSale = true;

The onSale property never existed before, but it does now!

What is console.log?

All that being said, what is console? What is log? And what is console.log?

  • console is an object.
  • log is the name of one of console's keys.
  • console.log represents the value that corresponds to the log key. It might seem odd that this value is a function, but remember, functions are data!

HTML-JavaScript Interaction

document.getElementById

With all of this explained, we can combine HTML and JavaScript together... sort of. Consider the following HTML and JavaScript in their respective tabs:


<!DOCTYPE html>
<html>
    <head>
    
    </head>
    <body>
        <p id="paragraph">JavaScript will change this text!</p>
    </body>
</html>
            

let paragraph = document.getElementById("paragraph");
paragraph.innerText = "new text";

This new function document.getElementById returns a JavaScript object representing an HTML element. More specifically, the JavaScript object represents an HTML element with the id attribute specified by its first argument. In the above example, we're getting the element with the id paragraph, which the p element in the body has. You can think of it like asking "hey, can you get me information about the element with the ID "paragraph"?"

In the second line, we change the value of the element's innerText property to "new text". The innerText property represents the text inside of an HTML element, and- when changed- the HTML element matches to update it. As such, all of this code together makes the HTML display the following:

new text

We can also read these properties just like any other property of a JavaScript object. So if we added console.log(paragraph.innerText) after our current code, the program would then output "new text" to the console.

Event Listeners

We had JavaScript control HTML in the last section. Now let's do the reverse with event listeners! Event listeners are simply functions that the browser calls whenever some event happens- for example if an element is clicked, the user scrolls the mouse, or a key is pressed. Here is an example:


<!DOCTYPE html>
<html>
    <head>
    
    </head>
    <body>
        <button id="click-me-button">Click me!</button>
    </body>
</html>
            

function clickHandler() {
    console.log("clicked!");
}

let button = document.getElementById("click-me-button");
button.onclick = clickHandler;

The HTML is standard stuff- a button with an id of click-me-button. The JavaScript starts with a function declaration that simply logs "clicked!" to the console. Then, we get the button with document.getElementById (notice, again, how the parameter to document.getElementById and the id attribute of the element are identical). Finally, we set the button's onclick property to the function. The browser knows to run the onclick function (if it exists) whenever the element is clicked, so clicking the button will now log "clicked!" to the console, because it runs clickHandler. Remember not to include parentheses after the function in this case, because we are referring to the function itself, not its return value (if any).

Note that the JavaScript can also be written like this to set button.onclick directly to the function without an intermediary. Also note the lack of a function name:


let button = document.getElementById("click-me-button");

button.onclick = function() {
    console.log("clicked!");
}

Input Elements

Inputs and the "Value" Attribute

<input> elements are among the most important of HTML elements. They allow the user to supply text, numbers, files, colors, and much more, data that can all be accessed from JavaScript.

As you might remember, <input>s have a value attribute containing whatever data has been supplied to them. This same data can be accessed from JavaScript with a property of the same name.

The "Change" Event

The change event is fired from an <input> element when the user exits its editable region after making a change to its value. The corresponding property for the change event is onchange (just like onclick for the click event). Remember that, in general, the property name will have "on" before the event name.

Type the following into JSFiddle, run it, and see what happens when you type into the text box:


<!DOCTYPE html>
<html>
    <head>
    
    </head>
    <body>
        <input value="change my text!" id="input">
    </body>
</html>
            

let input = document.getElementById("input");
input.onchange = function () {
    console.log(input.value);
}

The "Input" Event


let input = document.getElementById("input");
input.oninput = function () {
    console.log(input.value);
}

Now change the onchange to oninput (as shown above), re-run the program, and try typing again. Notice the difference in what appears in the console? The input event is triggered every time a character is entered.
